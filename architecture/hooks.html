    • Hooks

    Description

    A hook is just a callback function that gets executed when the data comes in or out the dispatcher for a given data flow. Multiple hooks can be tied to a single data flow, which in turn is tied to a single tag. Each hook can be set to run when the data comes in (input hook) or when the response goes out (output hook).

    The hooks have read-only access to the data/response objects.

    Hooks must extend the base Hook class.

    Exposed Methods

    • setup() - Initializes the hook.
    • run() - Executes the hook.

    Events

    Emits

    • onReload: Fired when the hook has been hot reloaded.
